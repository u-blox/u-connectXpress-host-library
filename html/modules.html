 ▼Communication with the moduleThe u-connectXpress host library allows easy integration of the u-connectXpress-capable modules into the customer's design using the command-callback framework
 ▼Sending data and controllingControlling the module is done using a combination of functions that are 1-to-1 mappings of the AT commands documented in the u-connect AT commands manual, and functions performing combination of commands. Responses from the module are optionally returned as callback
 Controlling the moduleFunctions that controls the module in a way that requires more than transmitting a simple AT command
 AT commandsFunctions that are a 1-to-1 mapping of AT commands
 Direct sending to the module independent of modeFunctions to invoke in order to send generic AT commands and data independent of if the module is in AT, DATA or Extended Data Mode (EDM)
 Direct sending to the module using EDMFunctions to invoke in order to send generic AT commands over the EDM AT channel, or data over a selected EDM data channel
 Direct sending to the module using AT and DATA modeFunctions to invoke in order to send generic AT commands in AT mode or data in DATA mode
 Receiving data from the moduleFunctions that are needed to be invoked at regular intervals in order to parse the incoming data and convert it to callbacks
 ▼CallbacksUpon reception of data from the module, optional callbacks previously registered by the application will be invoked
 Module CallbacksModule Callbacks are optionally invoked when the module changes state, not related to a specific AT command, URC or EDM Event, or as a result of functions involving several AT commands
 EDM CallbacksEDM Callbacks are optionally invoked when an Extended Data Mode (EDM) packet has arrived, or as a result of an EDM event
 Parser callbacksParser callbacks are optinally invoked when an AT OK or AT ERROR response is received, and/or upon the reception of parsed URC or unknown URC, independent of if the module is in EDM, AT or DATA mode
 ▼URC CallbacksURC Event Callbacks are optionally invoked upon reception of a known URC or command request
